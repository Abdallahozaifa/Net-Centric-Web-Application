Course Description:

The goal of the course is to introduce students to the most significant and exciting computing paradigmâ€”net-centric computing and service-oriented computing, which is expected to have an impact on all aspects of software construction at least as wide as that of object-oriented paradigm. This is a wide area and it is ever-evolving. We focus on the following aspects:

On the client: We will learn Cascading Style Sheets, JavaScript and AJAX to develop the client side of Rich Web Applications.

For client-server-communication: We will learn how to create and validate XML documents and use these as the primary language for transmission of data from the server to the client. We will also consider how JavaScript Object Notation (JSON) can sometimes be used as a viable alternative to XML for server to client data transmission.

On the server: We will learn about a variety of server-side technologies, including XML binding tools, JSP, and security. We especially focus on web services and learn how servers (or non-browser clients) may consume RESTful or SOAP-based web services.

Students will gain an in-depth understanding of the concepts and technical issues underpinning Web applications, and know how to use Web services technology effectively to design and develop complex net-centric systems for solving real-life problems.

Course Objectives

Upon completion of the course, students will know:

  1. Web MVC architecture
  2. Client-side techniques: CSS, Javascipt, jQuery
  3. Server-side techniques: JSP, XML, processing, EJB
  4. Namespaces in XML and their roles in SOA
  5. Fundamental XML Schema for describing document structures
  6. Fundamental WSDL for describing Web Services
  7. RESTful Web Services
  8. SOAP-based Web services and how to handle SOAP messages
  9. Steps to consume online web services
  10. How to deploy Web Services using Tomcat and Glassfish
  11. Web Services security, including user authentication, https
  12. Service-oriented architecture (SOA) for developing distributed software systems

Labs

The hands-on laboratory component concentrates mainly on the use of Java techniques to develop Web applications. The purpose of the lab assignments is to reinforce your understanding of the material taught in class, and encourage you to do research to broaden your knowledge on corresponding topics. The labs are group assignments. Each group should work as a team effectively to finish the assignments.


Labs are assigned on a weekly basis. Typically, the assignment for Fridayâ€™s lab will be posted to Angel on Thursday. Lab reports are due by midnight (11:59pm) Wednesday of the following week. Each team should have a person responsible for submitting your lab reports to the designated drop-box on Angel. Your submission should be in one zip file, following the pattern CSSE-WEB-Lab-X-Team-Y.zip to name the file, where X is the lab number (from 1 to 13), and Y is the team number assigned.


Lab solutions will be posted to Angel on Thursday. Labs that are turned in late (after midnight Wednesday) will be assessed with a penalty of 20%. However, if the lab assignment is turned in after the solution is posted, you will get the full penalty (0 point).


Course Project

The course project will consist of design and development of a web application using the techniques learned in this course. Each group functions as a team to work on the project. The purpose of this project is to help you gain in-depth experience with the development of Web Applications. You need to do your course project step-by-step as we introduce new topics along the semester. Your project status will be periodically checked and a grade may be assigned.
